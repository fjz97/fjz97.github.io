最近几场面试

2019年04月04日 3.8k 字 大概 15 分钟 随笔 随笔

第一家公司，车联网。面试时间定在了联系我的第二天上午10点。

面试官：介绍一下自己吧。
我：我是南京信息工程大学19届本科应届，专业是软件工程。大致情况就这样吧（完全不知道还有什么要介绍的）。
面试官：本科都开了哪些课程？
我：就是计算机专业的相关课程吧。数据结构、计算机组成原理、计算机网络、操作系统、编译原理等等，还有C/Java等计算机语言课程。
面试官：好的。我看你的项目经验里有“Android斗图APP”，能详细说下吗？
我：好的。这是一款图片APP，可以从服务器端获取到图片，用户可以浏览图片并保存在本地。另外，接入了QQ的SDK，可以将保存的图片分享给好友，所以说它是斗图APP。大致就是这么个功能的APP吧。
面试官：你在开发这款APP的时候遇到了什么难点呢？
我（想了想）：首先是腾讯的SDK吧，它的文档比较混乱，API的调用坑很多，花费了我很多时间。然后是图片OOM的问题，我做了分批次的请求处理。再然后就是图片下载的问题，因为腾讯的SDK出于安全考虑不允许用户分享网络图片，所以得先把图片下载到本地，但是我服务器下载速度比较慢，我就想了多线程下载等方法来解决，最后还是发现速度提升不大，估计得放到云服务器提供商提供的云存储（我忘了OSS怎么说了）等地方速度才会快点吧。
面试官：好的。那你有什么想要问我的呢？
我：我想问一下工作内容是什么呢？是应用开发吗？我希望能做应用开发。
面试官：有应用开发。
我：主要运用技术是什么呢？我看职位名称是Android多媒体开发，这个多媒体是用原生安卓开发呢，还是使用了网上的什么库或者框架呢？
面试官：都有。
我：那主要开发任务是什么呢？
面试官：像我们最近攻克的技术难点主要就有像分屏显示，balabalabalabala……（我真的记不得了）。
我（频频点头）：好的好的。
面试官：那我再问一个问题，你的理想薪资是多少？
我：我看你们招聘信息上最高开的是8k，我就恬不知耻的开8k了哈哈。
面试官：好的，没问题的。过几天我让我们人事再联络你吧。今天的面试就到这里吧。
我：好的，谢谢。

第二家公司，移动互联网公司，产品有点黑科技。面试定在了三天后的下午4点。

面试官：你先自我介绍一下吧。
我：我是南京信息工程大学本科大四应届，大概今年6月毕业（完全没吸取上次教训）。
面试官：有实习经验或者项目经验吗？
我：实习经验没有。您看过我的简历了吗，上面有写我做过的项目。
面试官：嗯，我看过了。你是有前端开发的经验是吗？
我：是的，不过是简单的前端项目，主要还是Android开发项目。
面试官：好的。那我就Android问些问题吧。
我：好的。
面试官：你知道Service吧，简单说下Service的种类吧。
我（有点懵逼，还以为要问项目相关问题）：Service是四大组件之一，又分为前台Service和后台Service（支支吾吾不知道还有什么值得说的了）。
面试官：你说到前台Service和后台Service，它们有什么区别呢？
我：后台Service就是直接开启的Service，前台Service可以通过Notification展示给用户并和用户做一些交互，前台Service的优先级较高，不易被杀掉。
面试官：你知道IntentService吗？
我：知道，但是我没用用过。好像是提供了一个异步回调方法，就不用自己重写异步了。
面试官：Service的onStartCommand方法会返回一个int类型的值，你知道它的作用是什么吗？
我：我不太了解。好像在刚学过的时候接触过，好像是表示该服务是否已经停止，比如返回-1表示服务已经停止，返回0表示没有停止。
面试官：好的。那说下Broadcast吧，Broadcast有哪些类型？
我：Broadcast分为静态注册的广播和动态注册的广播，但是我只用过动态注册的广播。静态注册的广播好像可以在应用未启动的时候接收到消息。
面试官：用过ListView吗？编写ListView有什么要注意的地方？
我：用过。主要还是要注意convertView的重用吧，还有就是使用ViewHolder。
面试官：是一个要注意的地方，其他还有吗？
我：一时想不起来了。我现在主要用RecyclerView，ListView用的少了。
面试官：那你说下RecyclerView的好处吧。
我（简直想给自己两巴掌）：RecyclerView就是回收机制更好一些，而且配合一些新的组件使用比较好（结结巴巴）。
面试官：那你说说什么时候该用ListView，什么时候用RecyclerView？
我：内容比较复杂且多的时候用RecyclerView，内容简单且少的时候用ListView。
面试官：了解事件分发机制吗？
我：是View的事件分发机制吗？不太了解。
面试官：知道Manifest吧，说下里面target-SDK和min-SDK两个标签的作用。
我：target-SDK就是编译使用的SDK版本，min-SDK就是兼容到的最小SDK版本。
面试官：好的。接下来问几个Java问题吧。
我（颤声）：好的。
面试官：能说下Java在什么情况下会出现内存泄漏吗？
我（awsl）：在使用线程的时候，线程运行完毕后没有回收线程。我一时只能想到这个了（想了半天）。
面试官：能说下二分查找的过程吗？
我：（balabala费力讲了半天，感觉自己语文水平是真的差，简单的原理讲也讲不明白）。
面试官（感觉有点无语，我对自己也很无语）：接下来说下HashMap吧。
我（绝望的一比）对不起，不太清楚。
面试官：好的。今天面试就到这里吧。
我：好的。

挂了电话我足足愣了有五分钟，真的是对自己失望透顶。仔细想想，真的都是基础的问题。居然被这些问题逼得如此窘迫。

这几天参加了两场校招，Android研发岗位寥寥无几，打印了十份简历，就投出去了一份，投的是一家手游公司。下午两点通知我参加宣讲会，三点半进行笔试，拿到试题，居然是公考题，无语，花了一个小时草草做完赶紧溜之大吉。吃晚饭的时候，居然收到消息通知我去参加面试，心里不由为之一喜，匆匆跑到大创，一进门先叫我等一会，排队面试。好不容易轮到我了，这一小时真是如坐针毡。一进门，我还没开口，面试官直接问我，你是面试Android开发的？不好意思，我们这Android位置满了，我看你简历上Java经历挺丰富的，不知道有没有兴趣做Java开发？气炸。真想给他来一套军体拳。白——等——了——

招聘信息岗位职责上直接写着：

掌握主流开源框架，如rxjava，okhttp，retrofit等

点开公司信息一看：

一家年轻的公司，致力于移动互联网APP开发，公司旗下有多款10万日活的APP，涉及阅读，电影，音乐，社区等。
所有产品都是自营，非外包。
涉及安卓，iOS，网站等。
公司福利好，今年年终奖半年以上的员工都有，年会还有2000元红包。
每天有下午茶，各种美食，反正张江周边都吃过了。
公司有小食吧，零食不断，半价全量供应。
公司内部还有健身房，全自动现磨咖啡机。
以下是废话：
balabala

Wow，这也太cool了吧，这绝对是一群geek，用着前沿的技术，做着有逼格的产品——这简直是我最理想的岗位啊！
抱着试一试的心态，我投了份简历。没想到很快就回复我了：

感谢你面试本公司的安卓岗位，为了节约你我的时间，在正式面试前我们发给你一个面试题，请带上编译成功后的作品的手机，来我司面试吧。
这是开源的项目，请把此项目在通过源码编译并成功地在你的手机上运行，请在“关于”页面加上你的名字的字样。
https://www.strongswan.org/

卧槽。这也太牛逼了吧。我喜欢。于是我开始看项目文档。

Native Build
Using a standalone toolchain from the Android NDK strongSwan can be built to run as console application/daemon on rooted Android systems.
Standalone Toolchain
Detailed instructions on how to create a standalone toolchain can be found in the Android NDK docs.
Something like the following will do:
$NDK/build/tools/make-standalone-toolchain.sh --arch=<arm|x86|mips> --platform=android-<API level> --install-dir=<path/to/toolchain>
For the instructions in the next section lets assume the toolchain was created like this:
$NDK/build/tools/make-standalone-toolchain.sh --arch=arm --platform=android-21 --install-dir=/tmp/strongswan-toolchain
Build strongSwan
Using the standalone toolchain created before strongSwan can be built from a source tree (or the repository) pretty much as usual. Therefore, refer to InstallationDocumentation for general instruction on how to build strongSwan from sources.
To cross-compile strongSwan for Android use the following when running ./configure (–host has to be adapted according to the selected architecture/toolchain):
export PATH=/tmp/strongswan-toolchain/bin:$PATH
./configure --host=arm-linux-androideabi <other options as needed>
Note: Depending on the API level, the older Linux headers included in the toolchain might make it necessary to add some additional headers in source:src/include or to revert the headers there to versions closer to the ones included in the toolchain.

要用NDK编译？我心里一紧。那就算编译完了我该怎么添加自己的名字呢？我对NDK一窍不通啊！不管，先动手做起来。
于是我开始下NDK，生成toolchain，用toolchain编译项目，一通操作，一会是找不到C编译器，一会又报GNU库文件丢失。整了一天毫无进展。绝望。天灭Windows。
第二天早上正发愁呢，没想到竟然在github上找到了项目的Android源码，clone，导入。居然没问题？！make project，报错，缺少运行时环境。解决方案：将项目官网提供的apk解压，把lib文件直接拷贝出来。完美解决！看了下源码，全是原生组件，居然都是我常用的。三下五除二加上自己的名字，搞定！

抱着忐忑的心理，把编译过后的截图发了过去，没想到对方真的邀请我面试了。于是第二天，我坐上了南京前往上海的高铁。面试时间定在了下午2点。
面试官：说下Android四大组件吧。
我：Activity，Service，Broadcast Receiver，Content Provider。
面试官：Activity的作用是什么？
我：Activity即为一个界面，用户的操作、与用户的交互都通过它完成。
面试官：Activity的生命周期是怎样的？
我：onCreate，onStart，onPause，onResume，onDestroy。还有一个和onStart对应的回调，我一时想不起来了。（现在想想我真是煞笔，start的反义词都不知道的吗）
面试官：onResume在什么时候被调用？
我：Activity从不可见到可见的时候。
面试官：对的。（我后来一查不对啊！）了解Android多线程吗？Android多线程主要用了什么类呢？
我：多线程？不是Thread类吗…
面试官：我是指！@#￥%（忘了他说的啥了）。
我：是指异步处理使用的类吗？主要是AsyncTask和Handler类，我主要用的是Handler类，AsyncTask类也是封装的Handler类，不过使用了线程池。
面试官：了解Android多进程通信吗？说下进程间通信的方式。
我：这个我没用使用过，不过我猜测可以通过Bundle传递数据。或者通过Service和Broadcast机制应该也可以实现。
面试官：好的。我没有问题要问了。
我：。。。（这就完了？）

第五家公司，移动互联网，D轮融资，毫无疑问是目前为止最好的机会。面试时间定在了第二天下午13点。
面试官：先做下自我介绍吧。
我：好的。我是南京信息工程大学本科大四在读，专业是软件工程。在校时开设了计算机网络，操作系统，编译原理等课程，也学习了C/Java等语言，取得了不错的成绩，专业排名在前10%。大概就是这样吧。（好久没面试了，感觉生疏了，项目经验和比赛经历什么的都没说）
面试官：好的。你说你校开设了计算机网络课程对吗？我问几个计算机网络问题吧。
我（一脸懵逼）：好的。
面试官：你知道http和https的区别吗？
我（黑人问号）：https应该是更安全的协议吧。具体不清楚，没有学习过。
面试官（循循善诱）：https做了加密处理，在https的包头里对参数做了处理。
我：哦。。。对不起，我不太清楚。
面试官：你知道HashMap吗？能不能说一下。
我：HashMap是一种数组+链表的数据结构。Java中HashMap使用了拉链法解决冲突。
面试官：那HashMap与HashTable有什么不同呢？
我：HashTable是线程安全的，它的方法中都加了synchronized锁。
面试官：那两者在访问上有什么区别吗？
我：HashMap访问速度应该更快吧，因为HashTable加入了同步，效率会降低。
面试官：访问速度？好吧。
我：。。。
面试官：讲一下快速排序的大致流程吧。
我：对不起，记不得了，我只知道是基于选择排序的策略。
面试官：说下Looper的作用。
我：不太清楚，我只知道它被用于消息处理机制，具体的API我没有研究过。
面试官：好的。你是刚开始找工作吗？我给你的建议，再做做功课。
我：好的，确实是准备不充分了。
面试官：你还有什么想要问的吗？
我：没有了。
挂了电话，我的表情应该是这样的↓

菜到被diss，太真实了。其实现在想想几个问题答的很不好，主要是一被问就慌了，让面试官给一点思考时间其实可以做的更好，比如HashMap和HashTable的区别，我觉得应该就同步这一点详细说一说，比如快排，我仔细想想应该也可以说出一点，再比如Looper的作用，虽然对原理知之不详，但是不断循环接受消息，维护消息队列等等应该好歹说一下，急着说不知道肯定是不行的。
